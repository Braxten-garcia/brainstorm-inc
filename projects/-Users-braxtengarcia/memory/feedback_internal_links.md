## Anchor text rules (per Google Search Central)

Write anchor text as naturally as possible. Google uses anchor text and its surrounding context to understand what the destination page is about.

Good anchor text is:
- Descriptive and reasonably concise
- Relevant to both the source page and the destination page
- Able to make sense when read out of context (e.g., a screen reader or link list)
- A natural phrase from body copy — never a heading (H1, H2, H3)

Avoid:
- Generic phrases: "click here," "read more," "learn more," "website"
- Keyword stuffing or excessively long anchor text
- Heading text as anchors — headings are structural, not contextual

**Why:** Google explicitly uses anchor text as a relevance signal for crawling and indexing. Bad anchor text misleads both users and Google about what the linked page contains.

## Semantic relevance rule

Only suggest a link if the destination page discusses the same topic as the anchor text and its surrounding content. Do not link on loose thematic connections (e.g., don't link "working document" to a Loop article unless the post is actually discussing Loop).

**Why:** Braxten flagged that loose links feel unnatural and don't serve the reader. Google's guidance confirms anchor text should reflect the destination — if you have to stretch to explain why it fits, it doesn't fit.

## Page type preference

Prefer middle and low-funnel pages (use cases, features, case studies, product pages, demo/contact) over other blog posts. Only suggest a blog link if the topic is a direct semantic match and no nav/product page covers it better.

**How to apply:** For every internal link candidate, check: (1) Is the anchor text natural body copy? (2) Does it make sense out of context? (3) Does the destination page discuss the same topic as the anchor and its surrounding sentences? (4) Is a nav/product page a better fit than a blog? Aim for 3–5 high-quality links over a longer list of loose ones.
